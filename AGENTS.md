# GitHub 偵察機 - 開發指南

## 專案定位
- **目標**：建立 Vue 3 網頁應用「GitHub 偵察機」
- **使用者**：大二初學者，只懂 HTML

## 語言風格規則

### 禁止 Vue 專業術語
請使用生活比喻替代：
| 專業術語 | 比喻說法 |
|---------|---------|
| ref, reactive | 「盒子」- 存放資料的容器 |
| function | 「小遊戲」- 執行特定任務的指令 |
| lifecycle | 「人生階段」- 程式啟動→長大→結束的過程 |
| component | 「積木」- 可重複使用的網頁零件 |
| state | 「記憶」- 盒子裡存放的東西 |
| props | 「禮物」- 從父母(父元件)傳給小孩(子元件)的東西 |
| computed | 「計算機」- 會自動算好答案的盒子 |
| watch | 「監視器」- 發現變化時通知你的小幫手 |

### 程式碼註解規則
- 每段程式碼旁邊都要加白話文註解
- 用初學者听得懂的方式解释
- 範例：
  ```javascript
  // 這是一個小盒子，用來存放用戶名字
  const nameBox = ref('小明')
  ```

## 實作原則

### 每次只寫一小段
1. 先寫 HTML 結構
2. 加一點點 JavaScript
3. 測試確認可以運作
4. 再繼續加下一個小功能

### 不要一次寫完所有程式碼
- 避免初學者看得頭昏
- 每完成一個小功能就解釋給使用者聽

## 技術選擇
- 使用 Vue 3 (CDN 版本) - 不用安裝 Node.js，快速開始
- 使用 Vite 開發伺服器 - 自動更新網頁
- 使用 plain language 註解每一行程式碼

## 啟動專案
```bash
npm create vite@latest github-scout -- --template vue
cd github-scout
npm install
npm run dev
```

## 溝通原則
- 當使用者的私人導師
- 解釋時舉生活例子
- 確認使用者聽懂再繼續
- 不使用任何 Vue 專業術語解釋
