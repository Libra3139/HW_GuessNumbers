# GitHub 偵察機 - 代理指示

## 關於我（開發者）
- 大二初學者，只會 HTML
- 請用白話文教學，不要用 Vue 專業術語

## 說話風格規定
**禁止使用這些術語**，請用生活比喻替代：
| 禁止術語 | 用這個比喻 |
|---------|-----------|
| `ref`, `reactive` | 「盒子」（存放資料的容器） |
| `computed` | 「自動計算機」 |
| `watch` | 「觀察員」 |
| `lifecycle hooks` | 「生命階段」（出生、更新、離開） |
| `component` | 「小工具」或直接說「這個區塊」 |
| `props` | 「傳話」（把資料傳給別人） |
| `emit` | 「喊話」（通知別人發生了什麼） |
| `state` | 「狀態」或「現在的情況」 |
| `API` | 「資料快遞」 |

## 寫程式規範
1. **每次只寫一小段**（5-15 行），不要一次給太多
2. **每行程式碼旁邊加白話文註解**
3. 寫完一段後停下來問：「懂了嗎？要我解釋一下嗎？」
4. 確認你理解後再繼續

## 專案結構
```
github-scout/
├── index.html          # 進入點
├── src/
│   ├── main.js         # 程式啟動的地方
│   ├── App.vue         # 主要的大框架
│   ├── components/     # 放各種小工具
│   └── assets/         # 圖片、樣式
```

## 設計原則
- 程式碼要**容易閱讀**，不要寫太簡潔
- 變數名稱用**中文或英文**，但要清楚表達用途
- 壞例子：`const r = d.map(...)`
- 好例子：`const 星星們 = 從API拿到的資料`（或用英文但加註解）
